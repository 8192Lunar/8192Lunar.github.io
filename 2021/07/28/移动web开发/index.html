移动web开发

Posted on 2021-07-28

一、基础

  1. 移动端浏览器内核主要都是由Webkit修改过来的，因此我们主要针对webkit内核进行兼容

  2. 视口

    视口就是浏览器显示页面内容的屏幕区域，视口可以分为布局视口、视觉视口和理想视口。

    布局视口（layout viewport）：一般移动设备的浏览器都默认设置了一个布局视口，用于解决早期PC端网页在手机上显示的问题，一般将这个视口分辨率设置为980px（会让元素看上去很小，可以通过手动来缩放页面）

    视觉视口（visual viewport）：用户能够看到的网站的区域，可以变化。

    理想视口（ideal viewport）：对设备来讲，是最理想的视口尺寸，需要手动添加meta视口标签通知浏览器操作，meta视口标签的主要目的是，布局视口的宽度应该与理想视口的宽度一致，即设备有多宽，布局的视口就有多宽

    标准写法：

name=”viewport” 视口标签属性

width=device-width 视口宽度为设备宽度

user-scalable=0(no)/[1(yes)] 是否允许用户缩放页面

initial-scale=1.0 初始页面缩放倍数

maximum-scale=1.0 最大缩放倍数

minimum-scale=1.0 最小缩放倍数

  1. 二倍图

    物理像素（屏幕分辨率）：屏幕显示的最小颗粒，是物理真实存在的。如：750*1334，即一行可以放750个最小颗粒，一列可以放1334个最小颗粒（像素点）

    一个px能显示的物理像素点的个数，称为物理像素比或屏幕像素比

    在PC端，1px等于1个物理像素，即物理像素比为1

    在iphone6/7/8，1px等于2个物理像素，物理像素比为2

    处理图片在移动端模糊问题：

    例:有一个50 50（css像素）的图片，在iPhone会放大2倍，变成100 100，所以我们采取，选择一个100 100的图片，然后手动将这个图片缩小为50 50

    即准备的图片比实际需要的图片大2倍，这就是2倍图

    处理精灵图时：

    首先把精灵图等比例缩放为原来的一半，然后再根据大小测量坐标，同时background-size也要写：精灵图原来宽度的一半

  2. 背景缩放

    background-size : 宽度 高度/cover/contain

    单位：像素/百分比（在只有一个数时等比例缩放，百分比单位则是相对父盒子来说的）

    cover：拉伸至整个盒子被完全覆盖，可能导致部分背景不全

    contain：等比例拉伸，当宽或高到达边界就不再拉伸，可能有部分空白区域

  3. 移动端开发选择

    主流方案：单独制作移动端页面（主流）/响应式页面兼容移动端

  4. 移动端技术解决方案

    CSS初始化：normalize.css

    官网地址：https://necolas.github.io/normalize.css/

    CSS3盒子模型：box-sizing

    box-sizing 属性:

    • content-box 是默认值。如果你设置一个元素的宽为100px，那么这个元素的内容区会有100px 宽，并且任何边框和内边距的宽度都会被增加到最后绘制出来的元素宽度中。即宽(高)度=width(height)+border+padding
    • border-box 告诉浏览器：你想要设置的边框和内边距的值是包含在width内的。也就是说，如果你将一个元素的width设为100px，那么这100px会包含它的border和padding，即内容区的实际宽度 = width - (border + padding)。

    注：border-box不包含margin

    总结：需要兼容，使用传统盒子模型；不考虑兼容，使用border-box盒子模型

    CSS3特殊样式：

    /*设置点击高亮的颜色*/

    -webkit-tap-highlight-color : color；`

    /*移动端浏览器默认外观在ios上加上这个属性才能给按钮和输入框自定义样式*/

    -webkit-appearance : none;

    /*禁用长按页面时的弹出菜单*/

    -webkit-touch-callout:none;

三、制作移动端页面布局

  1. 流式布局（百分比布局），也叫非固定像素布局

    通过盒子的宽度设置成百分比，根据屏幕的宽度来进行拉伸，不受固定像素的限制，内容向两侧填充。（但是高度和文字大小等大都是用px来固定）

  2. flex弹性布局

    （1）布局原理：通过给父盒子添加flex属性，来控制子盒子的位置和排列方式

    任何容器都可以为flex布局，当父盒子设置为flex布局后，子元素的float、clear和vertical-align属性都会失效

    ​ 容器：采用Flex布局的元素

    ​ 项目：容器的所有子元素成员，可以横向排列也可以纵向排列

    常用属性：

    ​ （2）父项常见属性(默认主轴就是x轴方向，水平向右；默认侧轴就是y轴方向，水平向下)：

    ​ flex-direction ：设置主轴方向

    | 属性值 | 说明 |
    | :————: | —————— |
    | row | 从左到右（默认值） |
    | row-reverse | 从右到左 |
    | column | 从上到下 |
    | column-reverse | 从下到上 |

    ​ justify-content ：设置主轴上的子元素排列方式

    | 属性值 | 说明 |
    | :———–: | ——————————————— |
    | flex-start | 从头部开始，若主轴是x轴，则从左到右（默认值） |
    | flex-end | 从尾部开始排列 |
    | center | 在主轴居中对齐（若主轴是x轴就水平居中） |
    | space-around | 平分剩余空间 |
    | space-between | 先两边贴边，再平分剩余空间 |

    ​ flex-wrap ：设置子元素是否换行，默认不换行，若装不开，会缩小子元素的宽度放到父盒子里

    | 属性值 | 说明 |
    | :—-: | ————– |
    | nowrap | 默认值，不换行 |
    | wrap | 换行 |

    ​ align-content ：设置侧轴上的子元素的排列方式（多行，子项出现换行的时期可以使用，单行时无效）

    | 属性值 | 说明 |
    | :———–: | ————————————– |
    | flex-start | 在侧轴头部开始排列（默认值） |
    | flex-end | 在侧轴尾部开始排列 |
    | center | 在侧轴中间显示 |
    | space-around | 子项在侧轴平分剩余空间 |
    | space-between | 子项在侧轴先分布在两头，再平分剩余空间 |
    | stretch | 设置子项元素高度平分父元素高度 |

    ​ align-items ：设置侧轴上的子元素排列方式（单行，只有一行元素时使用）

    | 属性值 | 说明 |
    | :——–: | ———————————— |
    | flex-start | 从上到下 |
    | flex-end | 从下到上 |
    | center | 挤在一起居中（垂直居中） |
    | stretch | 拉伸（默认值），但是子盒子不要给高度 |

    ​ flex-flow ：复合属性，相当于同时设置了flex-direction和flex-wrap

​ （3）子项常见属性：

​ flex：定义子项目分配剩余空间，用flex来表示占多少份数

单值语法: 值必须为以下其中之一:

  • 一个无单位数([]): 它会被当作flex:<number> 1 0; <flex-shrink>的值被假定为1，然后<flex-basis> 的值被假定为0。
  • 一个有效的宽度([width])值: 它会被当作 <flex-basis>的值。

  • 关键字none，auto或initial.

    双值语法: 第一个值必须为一个无单位数，并且它会被当作 <flex-grow> 的值。第二个值必须为以下之一：

  • 一个无单位数：它会被当作 <flex-shrink> 的值。

  • 一个有效的宽度值: 它会被当作 <flex-basis> 的值。

    三值语法:

  • 第一个值必须为一个无单位数，并且它会被当作 <flex-grow> 的值。

  • 第二个值必须为一个无单位数，并且它会被当作 <flex-shrink> 的值。

  • 第三个值必须为一个有效的宽度值， 并且它会被当作 <flex-basis> 的值。

    ​ align-self：控制子项自己在侧轴的排列方式，可覆盖align-items属性

    | 属性值 | 说明 |
    | :——–: | ———————————————————— |
    | auto | 设置为父元素的align-items的值，若没有父元素，则等同与stretch |
    | normal | 效果取决与当前布局模式：绝对定位布局中，normal在绝对定位的替代元素上表现为start，在所有其他绝对定位元素上表现为stretch，在绝对定位的静态元素上表现为stretch；flex布局中表现为stretch；在网格布局中表现为stretch，若是具有长宽比或内在大小的盒子，则表现为start；在块级和表格单元中无效 |
    | flex-start | 与侧轴头部对齐 |
    | flex-end | 与侧轴尾部对齐 |
    | center | 对齐到侧轴中部，如果该元素的尺寸大于 flex 容器，将在两个方向均等溢出。 |

    ​ order：定义子项的排列顺序（前后顺序），数值越小，排列越靠前，默认为0，注意：和z-index不一样，order是前后顺序，z-index是覆盖上下顺序

  1. less+rem+媒体查询布局

    （1）rem基础

    ​ rem ( root em ) 是一个相对单位，类似于em，em是相对于父元素字体大小；不同的是rem的基准是相对于html元素的字体大小，因此可以通过修改html里面的文字大小来改变页面中元素的大小

    比如，

    根元素（html）设置 font-size : 12px；非根元素设置width : 2rem；

    则换成px表示就是24px（12*2）

（2）媒体查询（Media Query）

  • 使用@media查询，可以针对不同的媒体类型定义不同的样式
  • @media可以针对不同的屏幕尺寸设置不同的样式
  • 当你重置浏览器大小的过程中，页面也会根据浏览器的宽度和高度重新渲染页面
  • 目前针对很多苹果手机、Android手机，平板等设备都用得到多媒体查询

语法规范：

  • 用@media开头 注意@符号

  • mediatype 媒体类型

    | 值 | 说明 |
    | :—-: | ———————————- |
    | all | 用于所有设备 |
    | print | 用于打印机和打印预览 |
    | screen | 用于电脑屏幕，平板电脑，智能手机等 |

  • 关键字，用于将媒体类型或多个媒体特性连接2到一起作为媒体查询的条件（ and not only ）

    and ：可以将多个媒体特性连接到一起，相当于“且”

    not ：排除某个媒体类型，”非“，可以省略

    only ：指定某个特定的媒体类型，可以省略

  • media feature 媒体特性 必须有小括号包含

    | 值 | 说明 |
    | :————: | ———————————- |
    | width | 定义输出设备中页面可见区域的宽度 |
    | min-width（≥） | 定义输出设备中页面最小可见区域宽度 |
    | max-width（≤） | 定义输出设备中页面最大可见区域宽度 |

    例1：

    上述例子，（页面拉伸）当屏幕尺寸小于800px时，屏幕会变成粉色；当屏幕尺寸（继续缩小）小于500px时，会变成紫色

    例2：rem+媒体查询实现元素动态大小变化

媒体查询还可以通过link引入不同的资源（样式文件）,即针对不同的屏幕尺寸调用不同的css文件

（3）less（.less文件）

​ Less(Leaner Style Sheets)是一门CSS扩展语言，也称为CSS预处理器，即Less是一门CSS预处理语言，它扩展了CSS的动态特性。

​ less变量：==@变量名：值;==

​ less编译：利用vscode的插件easy less

​ less嵌套：子元素的样式直接写到父元素中，如果要写伪类、伪元素、交集选择器，必须用 &

​ less运算：+ - * /

注：运算符中间左右必须有个空格隔开

​ 对于两个不同的单位的值之间的运算，运算结果的值取第一个值单位

​ 如果两个值之间只有一个值有单位，则运算结果就取该单位

​ （4）rem适配方案

​ 首先，根据设计稿与设备宽度的比例，动态计算并设置html根标签的font-size大小（媒体查询）；然后在CSS中，设计稿元素的宽、高、相对位置等取值，按照同等比例换算为rem为单位的值；

​ 技术方案：① less+媒体查询+rem；② flexible.js（移动端适配库）+rem；

公式如下：

页面元素的rem值 = 页面元素值（px）

/（屏幕宽度/划分的份数）

屏幕宽度/划分的份数 就是htmlfont-size的大小

或：页面元素的rem值 = 页面元素值（px）/html font-size字体大小

4.响应式布局

​ 原理：通过使用媒体查询针对不同宽度的设备进行布局和样式的设置，从而适配不同设备的目的。

响应式需要一个父级做完布局容器，来配合子级元素来实现变化效果。

在不同屏幕下，通过媒体查询来改变这个布局容器的大小，再改变里面子元素的排列方式和大小，从而实现不同屏幕下，看到不同的页面布局和样式变化。

设备划分 尺寸区间 宽度设置
超小屏幕（手机） < 768px 100%
小屏设备（平板） 768px ≤ 屏幕尺寸 < 992px 750px
中等屏幕（桌面显示器） 992px≤ 屏幕尺寸 < 1200px 970px
宽度设备（大桌面显示器） < 1200px 1170px
